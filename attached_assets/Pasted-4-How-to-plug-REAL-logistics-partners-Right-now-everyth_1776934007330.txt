4. How to plug REAL logistics partners
Right now everything is live-API-ready but falls back to simulation because no carrier keys are set. To go fully live, here's the play-by-play (Amazon-style):

A. Royal Mail (UK's biggest, best for parcels)
Sign up: https://developer.royalmail.net → register a "Click & Drop" or "Shipping API" app
They'll give you an API key + tracking URL prefix
Add to Replit Secrets: ROYAL_MAIL_API_KEY
Restart — every Royal Mail booking now hits their real API and prints a real label URL
B. DPD UK
Sign up: https://api.dpd.co.uk → "DPD Direct" partner account
They give you username + password (not OAuth)
Add Secrets: DPD_API_USERNAME, DPD_API_PASSWORD
Done — adapter logs in then books real shipments
C. WhatsApp business (so customers get real messages)
Set up Meta Business → WhatsApp Cloud API → get a permanent token + phone number ID
Secrets: WHATSAPP_TOKEN, WHATSAPP_PHONE_NUMBER_ID
Buyers get real WhatsApp on order paid
D. SendGrid email (real receipt emails)
Sign up sendgrid.com → API key
Secrets: SENDGRID_API_KEY, optionally SENDGRID_FROM_EMAIL
E. Other partners (Evri, FedEx, DHL, UPS, Stuart, Gophr)
Each one needs its own adapter file under server/shipping/adapters/. Royal Mail and DPD are templates — same pattern: bookShipment() returns {externalId, trackingNumber, trackingUrl, labelUrl}. I can wire any of these whenever you get keys — just say which one and add the secret.
